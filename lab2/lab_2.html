Creacin de procesos mediante fork

En Unix, un proceso es creado mediante la llamada al sistema fork(). El proceso que realiza la llamada se denomina proceso padre (parent process) y el proceso creado a partir de la llamada se denomina proceso hijo (child process). La sintaxis de la llamada, efectuada desde el proceso padre, es:

valor = fork()

La llamada fork() devuelve un valor distinto para los procesos padre e hijo: al proceso padre se le devuelve el PID del proceso hijo, mientras que al proceso hijo se le devuelve 0.

Las acciones que implican ejecutar un fork() son llevadas a cabo por el ncleo (kernel) del sistema operativo Unix. Dichas acciones son las siguientes:

  1. Asignacin de un hueco en la tabla de procesos para el nuevo proceso (hijo).
  2. Asignacin de un identificador nico (PID) al proceso hijo.
  3. Copia de la imagen del proceso padre al proceso hijo (con excepcin de la memoria compartida).
  4. Asignacin al proceso hijo del estado preparado para ejecucin.
  5. Devolucin de los dos valores de retorno de la funcin: al proceso padre se le entrega el PID del proceso hijo, y al proceso hijo se le entrega 0.

El siguiente cdigo, ubicado en el fichero forkprog1.c (carpeta de materiales), muestra el uso de la llamada a sistema fork():

#include <unistd.h>
            #include <stdio.h>
            #include <stdlib.h>
            
            int main() {
                pid_t rf;
                rf = fork();
                switch (rf) {
                    case -1:
                        printf("No he podido crear el proceso hijo \n");
                        break;
                    case 0:
                        printf("Soy el hijo, mi PID es %d y mi PPID es %d \n", getpid(), getppid());
                        sleep(20); //suspende el proceso 20 segundos
                        break;
                    default:
                        printf("Soy el padre, mi PID es %d y el PID de mi hijo es %d \n", getpid(), rf);
                        sleep(30); //suspende el proceso 30 segundos. Acaba antes el hijo.
                }
                printf("Final de ejecucin de %d \n", getpid());
                exit(0);
            }

Zonas de datos entre procesos vinculados por fork

Dos procesos vinculados por una llamada fork() (padre e hijo) poseen zonas de datos propias, de uso privado (no compartidas). Obviamente, al tratarse de procesos diferentes, cada uno posee un espacio de direcciones independiente e inviolable.

La ejecucin del siguiente programa, en el cual se asigna distinto valor a una misma variable segn se trate de la ejecucin del proceso padre o del hijo, nos permitir comprobar dicha caracterstica de la llamada fork().

forkprog2.c:

#include <unistd.h>
            #include <stdio.h>
            #include <stdlib.h>
            
            int main() {
                int i;
                int j;
                pid_t rf;
                rf = fork();
                switch (rf) {
                    case -1:
                        printf("\nNo he podido crear el proceso hijo");
                        break;
                    case 0:
                        i = 0;
                        printf("\nSoy el hijo, mi PID es %d y mi variable i (inicialmente a %d) es par", getpid(), i);
                        for (j = 0; j < 5; j++) {
                            i++;
                            i++;
                            printf("\nSoy el hijo, mi variable i es %d", i);
                        };
                        break;
                    default:
                        i = 1;
                        printf("\nSoy el padre, mi PID es %d y mi variable i (inicialmente a %d) es impar", getpid(), i);
                        for (j = 0; j < 5; j++) {
                            i++;
                            i++;
                            printf("\nSoy el padre, mi variable i es %d", i);
                        }
                }
                printf("\nFinal de ejecucion de %d\n", getpid());
                exit(0);
            }

En el programa anterior el proceso padre va dando valores impares a su variable i privada, mientras que el proceso hijo va dando valores pares a su variable i, tambin privada, que es diferente a la del proceso padre.

Para ejecutar este programa es preciso compilarlo primero:

            $ ./forkprog2

Ficheros abiertos entre procesos vinculados por fork

Los descriptores de ficheros (que indican qu ficheros tiene abierto un proceso) en el momento de hacer la llamada fork() son compartidos por los dos procesos que resultan de dicha llamada. Dicho de otra forma, los descriptores de ficheros en uso por el proceso padre son heredados por el proceso hijo generado.

En el siguiente programa, llamado forkprog3.c, dos procesos escriben distintas cadenas de caracteres en los mismos ficheros.

Pero antes de meternos con el programa en s es necesario aclarar un aspecto muy importante sobre el mismo. Ambos procesos (padre e hijo) realizan escrituras (llamadas al sistema write()) sobre los ficheros. Estas operaciones de Entrada/Salida deberan suspenderlos y, sin embargo, ambos realizan despus una llamada a usleep() precisamente para forzar su suspensin. La necesidad de utilizar usleep() proviene de cmo Linux implementa la llamada write(): en vez de realizar inmediatamente una escritura en disco (lo que s producira la suspensin del proceso), write() escribe los datos en una memoria intermedia (cach), que posteriormente ser volcada a disco. Linux realiza est accin con la intencin de agrupar varias llamadas a write() antes de acceder al disco, minimizando as el nmero de accesos. Por ello, el proceso no es obligatoriamente suspendido al utilizar write().

forkprog3.c:

#include <unistd.h>
            #include <stdlib.h>
            #include <stdio.h>
            #include <fcntl.h>
            
            int main() {
                int i;
                int fd1, fd2;
                const char string1[10]= "*********";
                const char string2[10]= "---------";
                pid_t rf;
                fd1 = creat("ficheroA", 0666);
                fd2 = creat("ficheroB", 0666);
                rf = fork();
                switch (rf) {
                    case -1:
                        printf("\nNo he podido crear el proceso hijo");
                        break;
                    case 0:
                        for (i = 0; i < 10; i++) {
                            write(fd1, string2, sizeof(string2));
                            write(fd2, string2, sizeof(string2));
                            usleep(1); /* Abandonamos voluntariamente el procesador */
                        }
                        break;
                    default:
                        for (i = 0; i < 10; i++) {
                            write(fd1, string1, sizeof(string1));
                            write(fd2, string1, sizeof(string1));
                            usleep(1); /* Abandonamos voluntariamente el procesador */
                        }
                }
                printf("\nFinal de ejecucion de %d \n", getpid());
                exit(0);
            }

            cat ficheroB

Ejecucin con espera

En el siguiente programa, llamado espe1.c, se emplea la llamada al sistema wait(). Esta llamada hace que el proceso que la invoca quede suspendido hasta que termine alguno de sus procesos hijos.

espe1.c:


            #include <unistd.h>
            #include <stdlib.h>
            #include <stdio.h>
            #include <wait.h>
            
            int main() {
                pid_t rf;
                rf = fork();
                switch (rf) {
                    case -1:
                        printf("\nNo he podido crear el proceso hijo");
                        break;
                    case 0:
                        printf("Soy el hijo, mi PID es %d y mi PPID es %d\n", getpid(), getppid());
                        sleep(10);
                        break;
                    default:
                        printf("Soy el padre, mi PID es %d y el PID de mi hijo es %d\n", getpid(), rf);
                        wait(0);
                }
                printf("\nFinal de ejecucion de %d \n", getpid());
                exit(0);
            }

Inicio de programas mediante exec

La llamada exec() sustituye el programa que la invoca por un nuevo programa. Mientras que fork() crea nuevos procesos, exec() sustituye la imagen en memoria del proceso por otra nueva (es decir, que sustituye todos los elementos del proceso: cdigo del programa, datos, pila y heap).

El PID del proceso es el mismo que antes de realizar la llamada exec(), pero ahora ejecuta otro programa. El proceso pasa a ejecutar el nuevo programa desde el inicio y la imagen en memoria del antiguo programa se pierde al verse sobrescrita. La imagen en memoria del antiguo programa se pierde para siempre, es decir, todo el cdigo que escribamos posteriormente a la ejecucin con xito de la llamada exec() ser inalcanzable.

La combinacin de las llamadas fork() y exec() es el mecanismo que ofrece Unix para la creacin de un nuevo proceso (fork()) que ejecute un programa determinado (exec()).

De las posibles llamadas tipo exec() (ver fichero en materiales llamadas_exec.pdf) usaremos para este apartado de la prctica la llamada execv(), cuya sintaxis es:

int execv(const char *filename, char *const argv[ ]);

Podemos seguir la ejecucin del proceso gracias a la orden ps, usndola justo antes y justo despus de la llamada execv(), comprobando as cmo se efecta la sustitucin de la imagen de memoria. El nuevo programa activado mantiene el mismo PID, as como otras propiedades asociadas al proceso. Sin embargo, el tamao en memoria de la imagen del proceso cambia, dado que el programa en ejecucin es diferente.

Manejo de seales

El programa signal.c hace uso de la llamada a sistema signal() para cambiar su comportamiento con respecto a las seales recibidas.

signal.c:

#include <unistd.h>
            #include <stdlib.h>
            #include <signal.h>
            #define VUELTAS 10000000000LL
            
            void confirmar(int sig) {
                char resp[100];
                write(1, "Quiere terminar? (s/n):", 24);
                read(0, resp, 100);
                if (resp[0]=='s') exit(0);
            }
            
            int main(void) {
                long long int i;
                signal(SIGINT, SIG_IGN);
                write(1, "No hago caso a CONTROL-C\n", 25);
                for (i=0; i<VUELTAS; i++);
                signal(SIGINT, SIG_DFL);
                write(1, "Ya hago caso a CONTROL-C\n", 25);
                for (i=0; i<VUELTAS; i++);
                signal(SIGINT, confirmar);
                write(1, "Ahora lo que digas\n", 19);
                for (i=0; i<VUELTAS; i++);
                exit(0);
            }

                        break;
                    default:
                        printf("\tSoy Abraham, mi PID es %d y el PID de mi hijo es %d. \n", getpid(), rf);

                        sleep(15);  //suspende el proceso 15 segundos.

                        sleep(1);
                }
                exit(0);
            }

Comunicacin mediante tuberas (pipes)

Las tuberas o pipes sirven para comunicar dos procesos, de tal forma que uno de ellos escribe en una tubera, mientras que el otro lee de ella. En Unix, adems de poder conectar dos procesos mediante una tubera usando el carcter |, tambin podemos crear un tipo de tubera conocida como pipe con nombre de forma explcita. Estas tuberas reciben el apelativo de con nombre porque, a diferencia de las creadas usando |, su existencia persiste ms all de la ejecucin de los procesos que la usan. Por tanto, deben ser borradas de forma explcita.

Como las pipes con nombre se comportan como una cola FIFO, tambin suelen recibir este nombre.

Crea una nueva pipe usando el siguiente comando:

mkfifo pruebafifo

Las pipes son manejadas por el sistema operativo como archivos, algo habitual (como seguro ya sabrs) en Unix. Puedes ver la pipe que acabas de crear haciendo un simple listado del directorio actual:

ls -al

Haz de la pipe la entrada estndar de un programa. Como, por ejemplo, de un sort:

sort < pruebafifo

Fjate en que sort por s mismo no hace nada. Abre otra ventana de terminal y escribe en ella lo siguiente:

Los pipes tambin se pueden usar dentro de un programa C. Estudia el siguiente ejemplo pipes.c.

#include <stdio.h>
            #include <stdlib.h>
            #include <unistd.h>
            #include <fcntl.h>
            
            int main(int argc, char* argv[]) {
              int fd[2];
              pid_t childpid;
              // Creamos el pipe, el array fd se usa para devolver 2 descriptores de fichero
              // que hacen referencia al principio y final del pipe
              pipe(fd);
            
              if ((childpid = fork()) == 0) {
                // El hijo va a ser ls
                // dup2 crea una copia del fichero apuntado por fd[1] usando el descriptor
                // de fichero STDOUT_FILENO. Es decir, tras dup2, la salida estndar de este
                // proceso coincidir con la salida del pipe y los dos pueden usarse
                // independientemente
                dup2(fd[1], STDOUT_FILENO);
                close(fd[0]); // ls no va a usar la entrada del pipe
                close(fd[1]); // ls va a usar como salida STDOUT_FILENO, que es una copia de fd[1]. Podemos cerrar esta copia
                execl("/bin/ls", "ls", "-l", NULL);
                perror("Fallo en ls");
              } else {
                // hacemos que la entrada estndar sea una copia del extremo de lectura del pipe
                dup2(fd[0], STDIN_FILENO);
                close(fd[0]); // cerramos fd[0], ya que ya hemos mapeado este extremo del pipe a STDIN
                close(fd[1]); // sort no va a hacer uso del extremo de escritura
                execl("/usr/bin/sort", "sort", "-k9", NULL);
                perror("Fallo en sort");
              }
              exit(0);
            }
